# Rest FastApi Swagger

Документация находится по пути: /docs

## Общее назначение
Проект представляет собой RESTful API для управления терминологическим глоссарием, разработанное на базе современного Python-фреймворка FastAPI. Система предназначена для хранения, получения, добавления, обновления и удаления терминов и их определений — например, в контексте выпускной квалификационной работы (ВКР), научного исследования или технической документации.

## Технологический стек 
- Backend: FastAPI (v0.121.3) — высокопроизводительный фреймворк для создания API
- ORM: SQLAlchemy (v2.0.44) — для взаимодействия с базой данных
- База данных: SQLite (встроенный файл glossary.db) — легковесное решение, подходящее для локальной разработки и небольших проектов
- Валидация данных: Pydantic (v2.12.4) — строгая типизация и автоматическая валидация входящих/исходящих данных
- Асинхронность: Поддержка через anyio, h11, uvicorn (ASGI-сервер)
- Тестирование: pytest + TestClient от FastAPI — полный покрытие основных сценариев (CRUD-операции, обработка ошибок)

#### Что реализовано?
Глоссарий поддерживает следующие операции:
- Получение списка всех терминов.
- Получение информации о конкретном термине по ключевому слову.
- Добавление нового термина с описанием.
- Обновление существующего термина.
- Удаление термина из глоссария.

Применяется Pydantic для валидации входных данных и формирования схем. 
Реализована автоматическая генерация статической документации. 
Для хранения используется SQLite.
Обеспечивается автоматическая миграция структуры данных при старте приложения.
